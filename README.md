<h2 data-start="836" data-end="886" class="">1 &nbsp;| &nbsp;Introduction</h2>
<p data-start="888" data-end="1142" class="">Samsung released “Art&nbsp;Mode” on their Frame‑series televisions to let owners display high‑resolution still images when the TV is idle. The <strong data-start="1026" data-end="1055">Samsung Art‑Mode Uploader</strong> (hereafter <em data-start="1067" data-end="1072">SAU</em>) is a command‑line tool that automates every part of that experience:</p>
<ul data-start="1144" data-end="1625">
<li data-start="1144" data-end="1210" class="">
<p data-start="1146" data-end="1210" class=""><strong data-start="1146" data-end="1159">Uploading</strong> images in either individual or batch (zip) form.</p>
</li>
<li data-start="1211" data-end="1280" class="">
<p data-start="1213" data-end="1280" class=""><strong data-start="1213" data-end="1236">Selecting / showing</strong> artwork instantaneously or on a schedule.</p>
</li>
<li data-start="1281" data-end="1368" class="">
<p data-start="1283" data-end="1368" class=""><strong data-start="1283" data-end="1304">Maintaining state</strong> so duplicates never waste TV flash storage or your bandwidth.</p>
</li>
<li data-start="1369" data-end="1471" class="">
<p data-start="1371" data-end="1471" class=""><strong data-start="1371" data-end="1390">Bulk management</strong> features such as matte tagging, quiet hours, daily quotas and remote clean‑up.</p>
</li>
<li data-start="1472" data-end="1625" class="">
<p data-start="1474" data-end="1625" class=""><strong data-start="1474" data-end="1501">Developer‑grade tooling</strong>—static typing, retries, concurrent preprocessing, CI hooks—so it’s robust enough for home labs <em data-start="1597" data-end="1602">and</em> gallery installations.</p>
</li>
</ul>
<p data-start="1627" data-end="1894" class="">This guide is intentionally exhaustive: every flag, environment variable, and edge‑case behaviour is covered. You can skim the <em data-start="1754" data-end="1769">Result Matrix</em> for a high‑level “what happens if I do X?” overview or dive deep via the <em data-start="1843" data-end="1862">Command Reference</em> and <em data-start="1867" data-end="1884">Advanced Topics</em> sections.</p>

</li>
<li data-start="11390" data-end="11428" class="">
<p data-start="11392" data-end="11428" class="">Back‑off pattern: 2 s → 4 s → 8 s.</p>
</li>
<li data-start="11429" data-end="11529" class="">
<p data-start="11431" data-end="11529" class="">After 3 failed attempts the program logs <code data-start="11472" data-end="11479">ERROR</code> and continues (bulk mode) or exits (single mode).</p>
</li>
</ul>
<h3 data-start="11531" data-end="11563" class="">9.3&nbsp;&nbsp;Concurrency Internals</h3>
<ul data-start="11565" data-end="11744">
<li data-start="11565" data-end="11646" class="">
<p data-start="11567" data-end="11646" class="">ThreadPoolExecutor only handles <strong data-start="11599" data-end="11608">local</strong> CPU tasks (hashing, Pillow resize).</p>
</li>
<li data-start="11647" data-end="11744" class="">
<p data-start="11649" data-end="11744" class="">Network uploads remain synchronous because the TV cannot accept parallel websocket transfers.</p>

<h2 data-start="13110" data-end="13178" class="">11 &nbsp;| &nbsp;Troubleshooting &amp; FAQ</h2>
<h3 data-start="13180" data-end="13267" class="">Q 1. <em data-start="13189" data-end="13265">“The first upload works but subsequent ones fail with <code data-start="13244" data-end="13262">401 Unauthorized</code>.”</em></h3>
<p data-start="13268" data-end="13473" class="">A: The TV revoked the websocket permission. Go to <em data-start="13318" data-end="13406">Settings → General → External Device Manager → Device&nbsp;Connection&nbsp;Manager → Device&nbsp;List</em>, remove the host entry, rerun <code data-start="13437" data-end="13444">check</code> and approve again on‑screen.</p>
<h3 data-start="13475" data-end="13544" class="">Q 2. <em data-start="13484" data-end="13542">“I renamed a file but it still says ‘Already uploaded’.”</em></h3>
<p data-start="13545" data-end="13680" class="">A: SAU matches by SHA‑1, not path. If the content is unchanged you don’t need to re‑upload. Touch or edit the image to change the hash.</p>
<h3 data-start="13682" data-end="13744" class="">Q 3. <em data-start="13691" data-end="13742">“Zip bulk upload returns 500 for large archives.”</em></h3>
<p data-start="13745" data-end="13865" class="">A: Many 2020‑era Frames have a 100 MB POST limit. Lower <code data-start="13801" data-end="13810">--batch</code> until each archive &lt; 80 MB or stick to single uploads.</p>
<h3 data-start="13867" data-end="13909" class="">Q 4. <em data-start="13876" data-end="13907">“My PNGs become JPEGs on TV.”</em></h3>
<p data-start="13910" data-end="14062" class="">A: The firmware recompresses everything internally to JPEG; matte overlays only work on that format. Quality is visually indistinguishable on 4 K panel.</p>
<h3 data-start="14064" data-end="14106" class="">Q 5. <em data-start="14073" data-end="14104">“Ruff lint fails on Windows.”</em></h3>
<p data-start="14107" data-end="14227" class="">A: Use <code data-start="14114" data-end="14139">pip install ruff==0.4.*</code> and ensure the script is inside UTF‑8 encoded repo; older CMD shells default to cp1252.</p>
